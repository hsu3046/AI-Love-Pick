# MatchMakerAI — 스코어링 로직 상세 문서

> **Version:** 2.0.0 | **Updated:** 2026-03-13
> 핵심 엔진: `src/engine/scoring.ts`

---

## TL;DR — 한 줄 요약

> **Phase 1(성향 6문항)** 이 "어떤 스타일의 AI가 맞는지" 성격을 파악하고,
> **Phase 2(실전 6문항)** 가 "실제로 뭘 추천할지" 범위·예산·조건을 결정한다.

### 쉬운 예시: 같은 앱, 완전 다른 결과

---

#### 🧑‍💻 예시 A: "꼼꼼한 개발자 민수"

| 질문 | 민수의 선택 |
|------|-----------|
| Q1 카톡 | B: 하나하나 정성껏 → **깊이 +1** |
| Q2 요리 | A: 레시피 따라함 → **현실 -1** |
| Q3 선물 | A: 리뷰 분석 → **논리 -1** |
| Q4 정리 | A: 체계적으로 → **체계 -1** |
| Q5 노래방 | B: 듣는 게 좋아 → **현실 -1** |
| Q6 유튜브 | A: 테크·IT → **체계 -1** + 🏷 `coding` |
| Q7 용도 | 코딩, 리서치 |
| Q8 빈도 | C: 업무 필수 |
| Q9 우선 | C: 품질 |
| Q10 기기 | B: PC |
| Q11 협업 | A: 혼자 |
| Q12 예산 | C: ~₩30,000 |

**결과:**
- 성향 점수: 깊이↑ 현실↑ 논리↑ 체계↑ → **Claude가 메인 LLM** (모든 축이 Claude와 일치)
- 보조 LLM: ChatGPT
- 코딩 도구: **Claude Code** (깊이+논리 축이 강하니까)
- 리서치 도구: **Consensus** (논리+깊이 축이 강하니까)
- 이미지/음악 → Q7에서 안 골랐으므로 **추천 0개**
- 추천 티어: Standard 또는 Pro

  - *Q2 +1, Q5 -1 → creative 상쇄 = 0

## Phase 1: 성향 질문 (Q1~Q6)

### 4축 스코어링 체계

각 축은 `-1` ~ `+1` 범위. A 선택 시 한쪽, B 선택 시 반대쪽으로 이동.

| 축 | -1 방향 | +1 방향 | 의미 |
|----|---------|---------|------|
| `speed_depth` | 속도 중시 | 깊이 중시 | 빠른 답변 vs 정확한 분석 |
| `real_creative` | 현실 중시 | 창작 중시 | 팩트 기반 vs 상상력 기반 |
| `logic_visual` | 논리 중시 | 감성 중시 | 데이터·분석 vs 직감·디자인 |
| `plan_flow` | 체계 중시 | 즉흥 중시 | 계획적 접근 vs 자유로운 흐름 |

---

### Q1. "카톡이 100개 쌓였다!"

| 선택 | 점수 변화 | 해석 |
|------|----------|------|
| A: 빠르게 훑어본다 | `speed_depth` **-1** | 속도 선호 → ChatGPT, Grok처럼 빠른 AI 선호 |
| B: 하나하나 정성껏 | `speed_depth` **+1** | 깊이 선호 → Claude처럼 정밀 분석 AI 선호 |

**영향:** 메인 LLM 결정에 핵심. Claude(+0.8)와 Grok(-0.9)의 speed_depth 차이가 가장 큼.

---

### Q2. "새로운 요리를 할 때 나는?"

| 선택 | 점수 변화 | 해석 |
|------|----------|------|
| A: 레시피 정확히 따라한다 | `real_creative` **-1** | 현실·정확성 선호 → Perplexity, NotebookLM |
| B: 감으로 즉흥 실험 | `real_creative` **+1** | 창작 선호 → Midjourney, Suno 등 창작 도구 |

**영향:** 이미지/음악/영상 서비스의 적합도에 직결. Midjourney(+0.9), Suno(+0.8) 등은 creative 축이 높음.

---

### Q3. "친구 생일 선물 고를 때"

| 선택 | 점수 변화 | 해석 |
|------|----------|------|
| A: 리뷰 분석 & 가성비 비교 | `logic_visual` **-1** | 논리 선호 → Claude, Cursor, Claude Code |
| B: 직감 선택 | `logic_visual` **+1** | 감성 선호 → Midjourney, Gemini 이미지 |

**영향:** 코딩 도구(Claude Code -0.9, Cursor -0.8)와 이미지 도구(Midjourney +0.9) 간의 분기점.

---

### Q4. "방 정리 스타일은?"

| 선택 | 점수 변화 | 해석 |
|------|----------|------|
| A: 구역별로 체계적으로 | `plan_flow` **-1** | 체계 선호 → Claude(-0.6), Cursor(-0.5), Google Opal(-0.7) |
| B: 눈에 보이는 것부터 | `plan_flow` **+1** | 즉흥 선호 → Grok(+0.6), Suno(+0.7), Seedream(+0.5) |

**영향:** 자동화 도구(Google Opal)와 즉흥적 창작 도구(Suno, Kling) 간의 분기.

|------|----------|-----------|------|

### Q6. "유튜브 알고리즘이 추천해주는 건?"

| 선택 | 점수 변화 | 특수 플래그 | 해석 |
|------|----------|-----------|------|
| A: 테크·과학·IT | `plan_flow` **-1** | 🏷 `coding` | 코딩 관심 → `usageNeeds`에 `coding` 자동 추가 |
| B: 디자인·예술·브이로그 | `plan_flow` **+1** | 🏷 `image` | 이미지 관심 → `usageNeeds`에 `image` 자동 추가 |
| C: 재테크·자기계발 | — (3개 선택지) | 🏷 `research` | 리서치 관심 → `usageNeeds`에 `research` 자동 추가 |

## Phase 2: 실전 질문 (Q7~Q12)

Phase 2의 답변은 직접 `PracticalProfile` 객체로 변환되어 서비스 적합도 계산에 사용.

---

### Q7. "AI를 어디에 쓰고 싶어요?" (멀티 셀렉트)

**가장 중요한 질문. 추천 서비스의 범위를 결정.**

| 선택 | usageNeed | 허용되는 서비스 카테고리 |
|------|-----------|----------------------|
| 글쓰기·번역·요약 | `writing` | `llm` |
| 이미지·디자인 | `image` | `image` |
| 코딩·개발 | `coding` | `coding` |
| 정보 검색·리서치 | `research` | `research` |
| 영상·음악 만들기 | `media` | `music`, `video`, `voice` |
| 업무 자동화 | `automation` | `automation` |
| 그냥 궁금한 거 질문 | `casual` | `llm` |

**게이팅 메커니즘:**
- **선택한 카테고리만** 추천 후보에 포함
- 선택하지 않은 카테고리의 서비스는 적합도 점수와 무관하게 **완전 차단**
- 예: `image` 미선택 → Midjourney, Seedream, GPT 이미지 등 모두 추천 불가
- Phase 1 플래그(`music`, `coding`, `image`, `research`)가 여기에 자동 추가됨

---

### Q8. "AI를 얼마나 자주 쓸 것 같아요?"

| 선택 | 값 | 적합도 보정 |
|------|----|-----------|
| A: 가끔 (주 1~2회) | `occasional` | 무료 서비스 **+0.1** 보너스 |
| B: 자주 (거의 매일) | `daily` | 보정 없음 |
| C: 업무 필수 | `heavy` | 유료 서비스 **+0.1** 보너스 |

**영향:** 추천 티어 결정에 사용. `heavy` + `quality` → Pro 티어 추천.

---

### Q9. "가장 중요한 건 뭐예요?"

| 선택 | 값 | 적합도 보정 |
|------|----|-----------|
| A: 비용 절약 | `cost` | 무료 서비스 **+0.15** 보너스 |
| B: 시간 절약 | `time` | 속도형(speed_depth < 0) 서비스 **+0.1** 보너스 |
| C: 결과물 품질 | `quality` | 프리미엄 서비스 **+0.15** 보너스 |

**영향:** Midjourney(premium)는 `quality` 선택 시 +0.15, `cost` 선택 시 무시. Seedream(free)은 `cost` 선택 시 +0.15.

---

### Q10. "AI를 주로 어디서 써요?"

| 선택 | 값 | 적합도 보정 |
|------|----|-----------|
| A: 모바일 | `mobile` | desktop 전용 서비스 **-0.15** 페널티 |

모든 서비스에 대해 아래 6단계 점수를 합산:

```
적합도 = ① 4축 내적
       + ② 예산 보정 (Q12)
       + ③ 우선순위 보정 (Q9)
       + ④ 빈도 보정 (Q8)
       + ⑤ 기기 보정 (Q10)
       + ⑥ 협업 보정 (Q11)
```

### ① 성향 매치 (4축 내적)

```
trait_score = user.speed_depth × svc.speed_depth
            + user.real_creative × svc.real_creative
            + user.logic_visual × svc.logic_visual
            + user.plan_flow × svc.plan_flow
```

범위: `-4.0` ~ `+4.0` (실제로는 `-2.0` ~ `+2.0` 근방)

## 최종 추천 결정

### 1. 유형 매칭 (재미 라벨용)

4축 점수를 8개 유형의 이상 좌표와 비교, **유클리드 거리 최소** 유형 선택.

```
distance = √( (user - type)² per axis )
```

이 유형은 결과 화면의 **성격 카드**에만 사용 (추천에는 영향 없음).

### 2. 메인 LLM 선택

LLM 카테고리(ChatGPT, Gemini, Claude, Grok) 중 **적합도 1위** = 메인.

### 3. 보조 LLM 선택

LLM 카테고리에서 **적합도 2위** = 보조.

### 4. 추가 서비스 선택

Q7에서 허용된 카테고리 내에서 **카테고리별 적합도 1위**만 추천.

→ 최대 추천 수 = 2(LLM) + Q7에서 선택한 카테고리 수

### 5. 추천 티어 결정

| 조건 | 티어 |
|------|------|
| 예산 ₩0 | Free |
| Pro 비용 ≤ 예산 + (heavy or quality) | Pro |
| Standard 비용 ≤ 예산 | Standard |
| 그 외 | Free |
