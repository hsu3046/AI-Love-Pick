# MatchMakerAI — 기술 결정 기록

> **Version:** 1.0.0 | **Updated:** 2026-03-12

---

## Decision 1: Vanilla TypeScript (프레임워크 미사용)

**결정:** React/Vue/Angular 없이 순수 TypeScript + DOM API로 구현

**근거:**
- 앱 규모가 작음 (5개 화면, 약 12개 소스 파일)
- 번들 사이즈 최소화 (런타임 의존성 `lucide` 1개)
- 간단한 화면 전환만 필요 (상태 관리 라이브러리 불필요)

**트레이드오프:**
- ✅ 빠른 로딩, 작은 번들 사이즈
- ⚠️ 코드 재사용성 제한 (`toPascalCase`, `injectIcon`이 `quiz.ts`와 `result.ts`에 중복)
- ⚠️ DOM 직접 조작으로 인한 XSS 리스크 (현재는 하드코딩 데이터만 사용하므로 무관)

---

## Decision 2: 유클리드 거리 기반 유형 매칭

**결정:** 4차원 좌표계에서 유클리드 거리(Euclidean Distance) 최소화로 유형 결정

**근거:**
- 각 유형이 4축의 이상 좌표(`traits`)를 가지므로 자연스러운 선택
- 가중치 없이 단순 거리 계산 → 직관적이고 디버깅 용이
- 모든 축을 동등하게 취급

**대안 검토:**
- 맨해튼 거리: 비슷하지만 대각선 방향 차이에 덜 민감
- 코사인 유사도: 방향만 비교하므로 Score 크기 정보 손실
- 가중 유클리드 거리: 축별 중요도 차등 부여 가능하지만 현재는 불필요

---

## Decision 3: 2-Phase 퀴즈 구조

**결정:** Phase 1 (성향 질문 6개) → Transition → Phase 2 (실전 질문 6개)

**근거:**
- Phase 1은 감성적/직관적 질문 → 사용자 참여도를 높이는 "훅" 역할
- Phase 2는 실용적 정보 수집 → 정확한 추천을 위한 데이터
- 중간 전환 화면으로 "결과가 나오고 있다"는 기대감 유발
- 질문 12개를 한꺼번에 보여주면 이탈률 상승 우려

---

## Decision 4: 3-Tier 플랜 시스템

**결정:** 무료 / 스탠다드 / 프로 3단계 플랜 자동 생성

**근거:**
- 사용자 예산 범위에 맞는 선택지 제공
- 무료 플랜 → AI 입문 장벽 낮춤
- 프로 플랜 → 전문가 니즈 충족
- 각 플랜은 메인 LLM + 보조 도구로 구성

**비용 추정 방식:**
- USD 가격에서 정규식으로 숫자 추출 (`priceLabel.match(/\$(\d+)/)`)
- KRW 변환: `× 1,400` (대략적 환율)
- 만원 단위 반올림 표시

---

## Decision 5: Lucide 아이콘 (런타임 SVG 생성)

**결정:** `lucide` 라이브러리의 `createElement` API를 사용하여 런타임에 SVG 아이콘 삽입

**근거:**
- kebab-case 아이콘 이름을 데이터에 문자열로 저장 가능
- `toPascalCase`로 변환하여 `icons` 객체에서 동적 조회
- Tree-shakeable하지만, `icons` 전체 import로 인해 실제로는 모든 아이콘 포함

**개선 가능성:**
- ⚠️ 현재 `icons` 전체를 import하므로 사용하지 않는 아이콘도 번들에 포함됨
- 개선 방안: 사용하는 아이콘만 명시적으로 import → 번들 사이즈 감소

---

## Decision 6: 파스텔 톤 디자인 시스템

**결정:** 웜 크림 배경(`#FFF8F0`) + 코럴 오렌지 액센트(`#FF8C6B`) + 파스텔 색상 팔레트

**근거:**
- 심리 테스트의 감성적 본질에 맞는 부드러운 디자인
- 과도한 정보 제공 없이 친근한 느낌
- 한국 사용자 대상 (한국어 UI, Pretendard 폰트)

---

## Decision 7: 모바일 퍼스트 (430px)

**결정:** `max-width: 430px` 기준 모바일 레이아웃, 데스크탑은 중앙 정렬 + 그림자

**근거:**
- 심리 테스트의 주 사용 환경: 모바일 (SNS 공유 통해 유입)
- 430px = 최신 iPhone 화면 너비 기준
- 데스크탑에서는 카드 형태로 표시하여 시각적 분리

---

## Decision 8: 개인화 Reasoning 시스템

**결정:** 단순 추천 나열이 아닌, 각 AI에 대해 "왜 이 AI인지" 이유를 제시

**근거:**
- 사용자가 추천을 신뢰하려면 근거가 필요
- 성향 축 점수와 AI 강점을 연결하여 개인화 메시지 생성
- 5단계 우선순위: 성향 매치 → 보조 AI → 용도 매치 → 플래그 기반 → 예산 최적
